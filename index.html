Beginner Getting started: Claude Code setup in 10 minutes
1

Install Node.js and Claude Code

Open Terminal (Mac: Cmd+Space → "Terminal"). Paste and run both commands.

# check node is installed (download from nodejs.org if not) node --version # install claude code globally npm install -g @anthropic-ai/claude-code
2

Get your API key

Go to console.anthropic.com → API Keys → Create Key. Copy it. Treat it like a password — never share it or put it in code you post publicly.

3

Authenticate and test

export ANTHROPIC_API_KEY="sk-ant-your-key-here" claude "say hello as a friendly clinician"
4

Run your first real clinical prompt

claude "Write a SOAP note for a 28-year-old runner with 3-week anterior knee pain worsening with stairs. No trauma. Full ROM. Pain with patellar compression."

Beginner Write any SOAP note from a clinical scenario in 60 seconds
1

The anatomy of a great clinical prompt

Include: patient demographics, chief complaint, timeline, relevant history, vitals, exam findings, and any labs or imaging. The more specific your input, the better the output.

2

Use this proven prompt template

claude "You are a clinical documentation assistant. Write a complete, professional SOAP note for the following scenario. Use standard medical formatting with S, O, A, P sections. Be thorough in the Assessment. Include a prioritized Plan with specific next steps. Patient: [demographics] CC: [chief complaint] HPI: [history] Exam: [findings] Data: [labs/imaging if any]"
3

Save to a file for EHR copy-paste

claude "[your prompt]" > note_$(date +%Y%m%d).txt cat note_*.txt

Ask Claude Code to write the script for you

claude "Write a Python script called prior_auth.py. Accept: patient name, ICD-10 code, requested service, and clinical justification as command-line arguments. Use the Anthropic SDK to generate a professional prior auth letter. Use argparse, include error handling, and save output to a .txt file."

Install dependencies and run it

pip install anthropic python prior_auth.py \ --patient "Jane D." \ --icd "M54.5" \ --service "Physical therapy x 12 visits" \ --justification "Lumbar radiculopathy, failed conservative care x 6 weeks"

Iterate with Claude

Paste the script back to Claude and say: "Make this more concise and add a section citing clinical necessity per Medicare LCD guidelines." Claude Code edits the file in place.

Compliance & Regulation

Legal landscape for clinical AI

The intersection of healthcare regulation and AI is evolving fast. This is what you need to know before you build, deploy, or use AI tools in clinical settings.

Foundation
HIPAA overview and what PHI means
Protected Health Information (PHI) is any health data that can identify a patient. HIPAA requires safeguards for PHI in all forms—written, spoken, and electronic. If an AI tool processes identifiable patient data, HIPAA applies. De-identified data (properly stripped of 18 identifiers per Safe Harbor method) is not PHI.
Foundation
Business Associate Agreements (BAAs)
A BAA is a contract required when a third party (like an AI vendor) handles PHI on your behalf. Without a signed BAA, using that tool with real patient data violates HIPAA. Major AI vendors (OpenAI, Anthropic, Google) offer BAAs for enterprise customers—but NOT for free-tier accounts. Always check before using.
Practical
FDA regulation — when AI becomes a medical device
If your AI tool diagnoses, treats, prevents, or mitigates disease, the FDA may classify it as a Software as a Medical Device (SaMD). Clinical decision support that merely organizes data (like a note generator) is usually exempt. But anything that makes clinical recommendations may trigger FDA oversight. Check the FDA's CDS guidance before deploying.
Practical
Malpractice and liability when AI is in the loop
Clinicians are legally responsible for all clinical decisions, even if informed by AI. "The AI told me to" is not a defense. If you use AI-generated content in patient care, you must review it for accuracy. Document your clinical reasoning. Malpractice insurers are watching this space—some policies explicitly exclude AI-related claims unless disclosed.
Advanced
State laws, FTC, and regulations beyond HIPAA
State laws vary widely. California (CMIA), Texas (HB 300), and others have stricter privacy rules than federal HIPAA. The FTC enforces consumer protection and can penalize misleading AI claims. If you market an AI tool, ensure transparency about what it does and doesn't do. Some states are drafting AI-specific healthcare laws—stay current.
Advanced
How to get institutional approval for AI tools
Most hospitals and clinics require vetting before clinical AI use. Steps: (1) Contact compliance or IT security, (2) Provide vendor documentation (SOC 2, BAA, security whitepaper), (3) Complete a risk assessment, (4) Get approval in writing. For academic projects, loop in your IRB early. Build relationships with compliance—they're gatekeepers but also allies.

Common Questions

Legal FAQ

The questions we hear most from students and early-career clinicians.

+
Can I use free AI tools (ChatGPT, Claude) for school assignments with fake patient cases?

Yes, as long as the cases are entirely fictional or properly de-identified. If your assignment uses real patient data from clinical rotations, you must strip all 18 HIPAA identifiers (name, DOB, MRN, dates, zip code beyond first 3 digits, etc.). When in doubt, make it up or ask your instructor.

+
My hospital doesn't have a BAA with any AI vendor. Can I still use AI tools at work?

Not with real PHI. You can use AI tools for non-patient tasks (e.g., drafting a protocol, summarizing research, learning). But entering real patient data without a BAA violates HIPAA. Talk to your compliance team about getting enterprise accounts with BAAs—or build internal tools that keep data on-premises.

+
If I build an AI tool for my clinic, do I need FDA approval?

It depends. Clinical decision support (CDS) that helps clinicians make decisions (not makes them for you) is usually exempt. If your tool only organizes information or automates documentation, likely no FDA review. But if it interprets data to diagnose, treat, or recommend interventions, it may be a Software as a Medical Device (SaMD) requiring FDA clearance. Review the FDA's 2022 CDS guidance or consult regulatory counsel.

+
Am I liable if AI gives me wrong information and I use it in patient care?

Yes. You are responsible for all clinical decisions, even if informed by AI. Standard of care requires you to verify AI-generated content before acting on it. If you blindly trust AI output and harm results, that's malpractice. Document your reasoning. Use AI as a tool, not a substitute for clinical judgment. Some malpractice carriers now ask about AI use—be transparent.

+
What's the difference between de-identified data and anonymized data?

De-identified data under HIPAA Safe Harbor removes 18 specific identifiers and has no reasonable basis to re-identify individuals. Anonymized data is irreversibly stripped of all identifying info. For practical purposes: if you remove name, DOB, MRN, exact dates, and zip codes (keep only first 3 digits), and there's no way to re-identify the patient, it's de-identified and no longer PHI. Use de-identified data for learning and demos.

+
Can I screenshot AI-generated notes and post them on social media to show what I built?

Only if fully de-identified or fictional. Even if you change the name, details like rare diagnoses, specific dates, or facility names can make a patient identifiable. Best practice: use completely made-up scenarios for demos. Never screenshot real patient notes, even with names redacted. One slip (a date, a hospital name, a unique case detail) can violate HIPAA and end your career.

Disclaimer: This content is educational and not legal advice. Regulations vary by jurisdiction and change frequently. Consult your institution's compliance office and legal counsel before deploying AI tools in clinical settings.
